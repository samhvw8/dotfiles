# Research Report Template

Save reports to `./plans/<plan-name>/reports/YYMMDD-<topic>.md`.

## Structure

```markdown
# Research Report: [Topic]

## Executive Summary
[2-3 paragraphs: key findings, recommendation, confidence level]

## Research Methodology
- Sources consulted: [number] across [languages]
- Date range: [earliest to most recent]
- Key search terms: [EN/CN/RU terms used]
- Retrieval tools used: [context7, gh, WebSearch, etc.]

## Key Findings

### Technology Overview
[What it is, current state, adoption trends]

### Best Practices
[Recommendations with explanations — note which community/language surfaced each]

### Security Considerations
[Vulnerabilities, mitigation strategies, recent CVEs if relevant]

### Performance Insights
[Benchmarks, optimization techniques, scale characteristics]

## Cross-Language Insights
[Where EN/CN/RU communities agree and where they diverge — and why]

## Comparative Analysis
[If applicable — comparison table of approaches/solutions]

## Implementation Recommendations

### Quick Start
[Step-by-step getting started]

### Code Examples
[Snippets with syntax highlighting]

### Common Pitfalls
[Mistakes to avoid — note which community documented each]

## Resources & References

### Official Documentation
- [Linked list]

### Community Resources
- [Forums, discussions, notable threads — across languages]

### GitHub Repositories
- [Top repos found via gh search]

## Unresolved Questions
[List anything that couldn't be conclusively answered]
```

## Formatting Rules

- Use code blocks with syntax highlighting
- Include mermaid or ASCII diagrams where helpful
- Sacrifice grammar for concision
- Include timestamp of when research was conducted
- Provide table of contents for reports exceeding 100 lines
- Conclude with specific, actionable next steps
